Research
Methods
Component audit across both team surfaces
Heuristic evaluation of existing loan flows
User testing with 3 design variations
Quantitative analysis of engagement & conversion
Key Findings
43%UI components were redundant across surfaces
35%Reduced cognitive load with winning variation
2×Faster iteration after combining teams
1stUsers preferred simplified single-view
Notes & Reflection
Gaps Identified
!Initial audit didn't account for edge cases in both component libraries
!Priority components weren't clearly defined before extraction began
!Testing with only 3 variations may have missed hybrid approaches
!Post-merge governance for the combined system was undefined
How I'd Improve Today

I'd start with a shared component taxonomy before any experimentation — Audit + Simplify + Align + Audit as a continuous cycle. Ongoing governance rituals prevent divergence. I'd also measure beyond conversion: trust, clarity, and confidence.
